# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities privately to project maintainers.
Do not open public issues for security-sensitive reports.

Include:

- affected version(s)
- impact summary
- reproduction steps or proof of concept
- suggested mitigation if available

## Response Process

Maintainers will:

- acknowledge receipt
- validate impact and scope
- prepare and release a fix
- publish advisory notes when appropriate

## Scope

Security reports are especially relevant for:

- runtime event handling
- DOM selector and mapping behavior
- script integration boundaries
- dependency vulnerabilities in release artifacts
